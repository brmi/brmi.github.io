Daily Bruin Print Archive Data Collection

The goal of the site is to have one place where a reader can easily access the day's paper from any day in the Daily Bruin's history. The tricky thing is, some of the archive.org reels cover multiple years, while others cover only a few months; and there's no way to know the start or end dates without going through them manually. Additionally, searching for a date in the middle of the reel, which contains different newspapers of different page lengths, is a challenge. The Google Drive folder also grows nightly.

This specific sub-project has to do with identifying dates on a specific PDF page. OCRing the PDFs doesn't seem to be a reliable way to identify the dates on the page. The only way to do this is manually. The goal of this project is to have people manually input the date with the PDF page shown on the screen. With over 100 years worth of newpapers, we want as many people helping to manually add the dates as possible. To do this, we need a user friendly interface.

SPECIFIC FEATURES
  • Let user enter date in calendar module so input formats will be the same.
  • Let user's double check inputs from other users past inputs
